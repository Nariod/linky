[✅] 1.6.6  CLI — RÉORGANISER LE HELP DU MENU LINK (UX)
         Problème : les commandes Windows sont dispersées à deux endroits
         dans print_link_help() :
           - "Execution" → cmd, powershell (avec "(Windows only)")
           - "Windows only" → integrity, inject
         Un opérateur sur un link Linux voit des commandes inutiles mélangées.
         FIX (2 améliorations) :
         a) print_link_help() prend la plateforme en paramètre et n'affiche
            les commandes Windows (cmd, powershell, integrity, inject) que
            si le link est Windows. cmd et powershell rejoignent la section
            "Execution" (affichée seulement sur Windows). integrity et inject
            vont dans une section "Windows" unique (au lieu de l'ancienne
            section "Windows only" séparée).
         b) Le prompt passe de "link-1> " à "link-1|lnx> " ou "link-1|win> "
            pour que l'opérateur voie toujours la plateforme sans taper info.
         Fichier : server/src/cli.rs
         Difficulté : faible (~15 min)
